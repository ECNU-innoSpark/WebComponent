## Agent And Multimodal Patterns

### 1. InnoSpark 优先支持的 demo 类型

| Demo 类型 | 推荐模板 | 必备组件 |
| --- | --- | --- |
| 个性化知识点讲解 | 单 Agent Demo | 对话区、知识来源、概念卡、练习入口 |
| 引导式学习辅导 | 单 Agent Demo / 多步骤页 | 目标卡、过程提示、阶段反馈、下一步任务 |
| 跨学科教案生成 | 单 Agent Demo / 工作流页 | 输入表单、生成结果、资源引用、导出区 |
| 情景化智能出题 | 单 Agent Demo | 条件配置、题目列表、难度标签、答案与解析 |
| 多模态课堂分析 | 多模态 Studio | 资源条、时间轴、摘要卡、证据定位 |
| ELMES 课程评测 | 数据报告模板 | 指标卡、维度图表、证据样本、建议区 |
| 多代理协作 | 工作流模板 | 节点图、步骤状态、结果总线、协作记录 |

### 2. Agent 核心组件

#### Agent Card

最低字段：

- 名称
- 一句话价值
- 场景标签
- 输入模态标签
- 输出模态标签
- 适用用户
- 试用按钮

#### Capability Tag / Scene Tag

统一使用胶囊标签表达：

- 学段：小学 / 初中 / 高中 / 高校 / 职教
- 对象：学生 / 教师 / 教研 / 管理者
- 场景：备课 / 教学 / 作业 / 评测 / 研究 / 运营
- 模态：文本 / 图像 / 音频 / 视频 / 文档 / 结构化数据

#### Chat Message

- 用户消息和系统消息要区分
- 系统回复要支持段落、列表、引用、来源块
- 不做娱乐化聊天风
- 结论要区分“事实 / 推断 / 建议”

#### Citation Block

- 编号
- 来源名称
- 时间或版本
- 摘要
- 跳转入口

#### Insight Card

- 结论
- 可信度
- 依据
- 建议操作

#### Workflow Board

- 显示输入、动作、输出、状态
- 连接线表达依赖关系，不做纯装饰
- 可用横向流、纵向流、泳道布局

### 3. Demo Shell 统一规则

所有 agent demo 默认使用 6 块结构：

1. 场景标题与适用对象
2. 输入方式与资源来源
3. 核心输出
4. 证据与依据
5. 可继续操作
6. 风险提示或适用边界

### 4. 多模态资源组件

#### Multimodal Asset Card

| 资源类型 | 最低展示字段 |
| --- | --- |
| 文档 | 标题、页数、来源、摘要 |
| 图片 | 缩略图、说明、识别结果 |
| 音频 | 时长、说话人、转写摘要 |
| 视频 | 封面、时长、关键片段 |
| 结构化数据 | 数据集名、字段数、更新时间 |

#### Source Panel / Evidence Drawer

- 支持折叠
- 支持编号引用
- 支持跳转到页码、时间点或片段
- 支持高亮摘录

#### Knowledge Source Stack

- 展示来源类型、数量、可信级别、更新时间
- 支持“知识库 / 外部文档 / 教材 / 课堂记录 / 多媒体资源”分类

### 5. 多模态教学 Studio 规则

- 左侧：资源流或资源树
- 主区：当前资源预览
- 右侧或下方：摘要、证据、建议
- 时间轴必须可点击定位
- 分析结果必须能回跳到资源原位置

### 6. 课堂观察与评测面板

#### Rubric / Evaluation Panel

- 维度
- 分值
- 等级
- 描述
- 建议

#### Classroom Observation / Evaluation Panel

- 支持按教师、课程、班级、时间切换视角
- 观察维度与课堂片段联动
- 报告页与交互页尽量共用同一评价组件体系

### 7. 设计注意点

- 不要只做聊天框
- 不要只给结论不给证据
- 不要让多模态资源与分析结果分离得太远
- 不要让老师在多个页面之间来回找来源和样本
